# feat: 新增市场格局策略分析

## 需求理解
提交当前工作区全部变更：新增市场格局风向标策略分析能力，打通 AkShare ETF 实时行情、后端策略接口、策略智能体工具调用和前端策略页面展示。

## 改动文件列表
- `app/modules/strategy/`：新增策略模块的路由、Schema 和服务实现。
- `app/api/router.py`：注册策略模块 API 路由。
- `app/infrastructure/clients/akshare_client.py`：新增 ETF 实时行情查询缓存与清理能力。
- `app/modules/agent/`：新增盘中观察员、纪律风控官内置智能体，接入市场格局工具和运行时分发。
- `web/src/pages/strategies/`：新增市场格局风向标页面和前端配置映射。
- `web/src/api/strategy.ts`：新增策略分析接口类型与请求封装。
- `web/src/utils/route.ts`、`web/src/utils/navigation.ts`、`web/src/App.tsx`、`web/src/layouts/AdminLayout.tsx`：支持嵌套路由、菜单分组和策略页面入口。
- `web/src/App.css`：新增策略页面布局、四象限、观察点和纪律面板样式。
- `tests/`、`web/scripts/`、`web/package.json`：新增或更新后端与前端测试。
- `pyproject.toml`、`uv.lock`：新增 `mineru[all]` 依赖并更新锁文件。

## 关键实现说明
- 后端策略服务基于核心 ETF 涨跌幅判断四象限市场剧本，并派生盘中观察点、纪律建议、风险等级、ETF 搜索结果和智能体工具上下文。
- AkShare ETF spot 查询加入 20 秒进程内缓存，减少同一页面和 Agent 多次请求造成的重复外部调用。
- Agent 工具网关新增市场格局观察、纪律建议、上下文查询和 ETF 搜索工具，并为两个策略智能体配置独立 persona prompt 与运行图分发。
- 前端策略页并行加载盘中观察和纪律建议，展示主线风向标、四象限剧本、观察板块、风险纪律和自动刷新状态。
- 导航工具支持菜单路由与叶子页面路由拆分，保证嵌套菜单可选中且实际页面路由可正常注册。

## 风险与兼容性影响
- `mineru[all]` 带来较大的依赖锁文件变化，需关注安装体积、跨平台解析和 CI 缓存耗时。
- AkShare 字段名和实时行情可用性依赖外部数据源，异常场景通过 BadGateway 返回，仍需关注线上接口波动。
- 前端新增嵌套路由结构，需确认现有一级菜单页面仍能正常跳转和选中。
- 投资策略内容仅作信息分析，智能体 prompt 已加入不构成投资建议和禁止高风险表达约束。

## 回滚建议
- 如策略接口或 AkShare 数据源不稳定，可回滚 `app/modules/strategy/`、路由注册和 Agent 工具接入，并移除前端策略入口。
- 如仅前端展示异常，可保留后端接口，回滚 `web/src/pages/strategies/`、导航配置和相关 CSS。
- 如依赖安装出现问题，可优先回滚 `pyproject.toml` 与 `uv.lock` 中 `mineru[all]` 相关变更，再单独评估依赖拆分。
